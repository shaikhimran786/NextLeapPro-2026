Context:
We are building Next Leap Pro, a community-driven platform where users can join paid events, workshops, and sessions. Currently, the event payment flow is integrated with Cashfree.

Objective:
Replace Cashfree with Razorpay for all event-related payments and ensure a consistent payment experience across the entire application for users in India.

🎯 Requirements
Remove Cashfree Integration (Events Only)
Identify all event-related payment flows using Cashfree
Remove or disable Cashfree SDK, APIs, and configs specifically for event payments
Ensure no breaking changes for other modules (if Cashfree is used elsewhere)
Integrate Razorpay for Event Payments
Use Razorpay as the primary payment gateway
Implement Razorpay Checkout for frontend payment flow
Create backend API to generate Razorpay order (/api/payments/create-order)
Pass required details:
amount
currency (INR)
event_id
user_id
Standardise Payment Flow Across Application
Ensure all event payments follow the same structure:
Create order from backend
Open Razorpay checkout
Handle success/failure
Verify payment signature on backend
Maintain a unified payment service layer (avoid scattered logic)
Payment Verification & Security
Implement Razorpay signature verification on backend
Store:
razorpay_payment_id
razorpay_order_id
razorpay_signature
Mark event registration as “confirmed” only after verification
Database Updates
Update payment table schema:
provider = "razorpay"
status (created, success, failed)
Map payments to event registrations
Frontend UX Consistency
Keep payment UI clean and minimal
Reuse same payment button/component across:
Event pages
Checkout screens
Show clear states:
Processing
Success
Failed
Region-Based Logic (India Focus)
Default payment gateway for India users = Razorpay

Add config:

PAYMENT_PROVIDER=razorpay
REGION=india
Keep system extensible for future international gateways

Environment Variables
Add:

RAZORPAY_KEY_ID=
RAZORPAY_KEY_SECRET=
Testing
Test successful payment flow
Test failed payment scenario
Test signature verification failure
Ensure event access is granted only after success
🚀 Expected Outcome
All event payments use Razorpay
Clean, consistent, and reusable payment flow
Secure backend verification
Better trust and familiarity for Indian users
💡 Implementation Insight

Right now, this is not just a payment change.

This is about trust + consistency.

In India, users are already comfortable with Razorpay. The smoother and more predictable your payment flow is, the higher your event conversion will be.
