### 1. Avatar Image Upgrade

**Current Issue:**
Testimonials are using initials (e.g., “AP”), which feels generic and low-trust.

**Implementation:**

* Add/update testimonial data structure:

```ts
{
  name: string,
  role: string,
  title: string,
  testimonial: string,
  avatarUrl?: string,
  gender?: "male" | "female"
}
```

**Avatar Logic:**

* If `avatarUrl` exists → show real image
* Else → use realistic human avatar generator (not cartoonish)

**Use natural human avatars:**

* Prefer sources like:

  * [https://randomuser.me/api/portraits/men/](https://randomuser.me/api/portraits/men/)
  * [https://randomuser.me/api/portraits/women/](https://randomuser.me/api/portraits/women/)

**Example fallback logic:**

```ts
const avatar = avatarUrl 
  ? avatarUrl 
  : gender === "female"
    ? `https://randomuser.me/api/portraits/women/${index + 10}.jpg`
    : `https://randomuser.me/api/portraits/men/${index + 10}.jpg`
```

**UI Update:**

```tsx
<img
  src={avatar}
  alt={name}
  className="w-12 h-12 rounded-full object-cover shadow-sm"
/>
```

**Constraints:**

* Keep avatars realistic (avoid illustrations/cartoon avatars)
* Maintain consistent size and alignment
* Ensure fast loading
