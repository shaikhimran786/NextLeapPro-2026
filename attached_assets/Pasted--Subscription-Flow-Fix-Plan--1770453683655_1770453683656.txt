 Subscription Flow Fix Plan                                                                                                   │
│                                                                                                                              │
│ Context                                                                                                                      │
│                                                                                                                              │
│ The /pricing page subscription flow is broken end-to-end. Users who click "Subscribe Now" while logged out are redirected to │
│  login, but after logging in they land on /dashboard instead of returning to complete checkout. Additionally, the checkout   │
│ API trusts a client-sent userId (security vulnerability), duplicate clicks can create multiple orders, and the Razorpay      │
│ integration is missing error handling for failed payments. This plan fixes all blockers and hardens the flow to              │
│ production-grade quality.                                                                                                    │
│                                                                                                                              │

│ Verification                                                                                                                 │
│                                                                                                                              │
│ 1. Auth redirect flow: Click Subscribe on a paid plan while logged out. Verify redirect to                                   │
│ /auth/login?redirect=/pricing?plan=X&checkout=true. Log in. Verify auto-redirect back to pricing and Razorpay checkout opens │
│  automatically.                                                                                                              │
│ 2. Register flow: From the login page (with redirect params), click "Sign up". Register. Verify direct redirect to pricing   │
│ page with auto-checkout (no login step).                                                                                     │
│ 3. Security: Call POST /api/subscriptions/checkout without a session cookie — expect 401. Call with valid session but forged │
│  userId in body — expect the server to use the session user, not the body user.                                              │
│ 4. Double-click: Rapidly click Subscribe multiple times. Verify only one checkout session is created (check DB or network    │
│ tab).                                                                                                                        │
│ 5. Payment failure: Use Razorpay test mode with a failing card. Verify toast error appears in the app.                       │
│ 6. Button states: Click Subscribe on one plan. Verify all other plan buttons are disabled.                                   │
│ 7. Script dedup: Complete checkout, navigate back, try again — inspect DOM for duplicate script tags.                        │
│ 8. Dashboard: After successful payment, verify dashboard shows the correct active plan immediately.
